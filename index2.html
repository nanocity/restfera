Recursos

  • Todo es un recurso
  • Tienen una URN únivoca
  • Las operaciones se realizan sobre ellos
  • Tienen sentido en la lógica de negocio de la aplicación

Cliente - Servidor

  • Separación conceptual
  • Permite el desarrollo independiente
  • Consumidor - Servicio

Stateless

  • El servidor no guarda estados
  • Peticiones/Respuestas con toda la información

Caché

  • Mejora el rendimiento
  • Se controla mediante E-Tags
  • Cabecera: If-None-Match

Interfaz Uniforme

  • Acceso a la informacion de forma estándar
  • Separación entre lógica de aplicación y lógica de negocio
  • Facilita el desarrolo de clientes genéricos

Sistema de Capas

  • Permite agregar capas nuevas de forma transparente
  • Una capa solo es consciente de la capa siguiente
  • Simplifica arquitecturas distribuidas

Código bajo demanda

  • Opcional
  • Uso de plugins
  • Permite delegar logica en los clientes
